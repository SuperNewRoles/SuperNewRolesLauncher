## Welcome!
なにもなしにプルリクエストを送っても問題ありません。
あなたのプルリクエストを歓迎します。特別な許可や条件は不要です。

また、プログラムを書けない場合はIssueを作成するか公式Discordサーバーで改善点/バグの報告を行なうことでも貢献できます。

### 1. いきなりPR送信OK
最小限の修正でも歓迎します。
- 軽微な修正
- タイポや文面修正
- リファクタリング
- 機能改善
- アクセシビリティやUX改善

### 2. AIエージェント使用OK
ChatGPT・Claude・Github Copilotなど、AIエージェントによる支援を受けたコーディングを歓迎します。

### 3. PRのレビュー方針
基本的に、雑に見える大量の低品質PRを連投するレベルでは限り、どんなPRでも確認します。
無理にハイレベルな提出でなくても、丁寧に書かれたものならフィードバック付きで受け取ります。

提出前のおすすめ:
- 変更範囲を小さくする
- 必要ならテスト・動作確認手順を添える
- 無関係な整形のみの変更は控える
- 既存挙動への影響やリスクを明記する
- PR要約・確認ポイントを簡潔に書く

開発者のレビューはすぐにはできない可能性があります。ご了承ください。

### 4. PR提出チェック項目（任意）
- どのIssueに対応した変更か
- 解決しようとしている問題は何か
- 変更点の要約
- 動作確認手順
- 互換性・副作用の有無
- 必要に応じた画像やログ

### 5. バグ・改善提案を書く
再現手順・環境（OS/バージョン/ロケール）・期待挙動・実際挙動を含めてください。  
これがあると、修正までの時間を短縮できます。
---
気軽に送ってください。敷居の高さを気にせず、提案・PR・Issue・Discordでの報告を歓迎します。

# English
## Welcome!
It is perfectly fine to send a pull request without any prior notice. 
Your pull requests are welcome; no special permission or prerequisites are required.

If you are not a programmer, you can still contribute by creating an Issue or reporting bugs and improvements on the official Discord server.

### 1. Feel free to send PRs immediately
Even the smallest fixes are welcome.
- Minor fixes
- Typos or wording corrections
- Refactoring
- Feature improvements
- Accessibility and UX improvements

### 2. AI Agents are allowed
We welcome coding assisted by AI agents such as ChatGPT, Claude, and GitHub Copilot.

### 3. PR Review Policy
In principle, we will review any PR, provided it is not a barrage of low-quality, "spammy" submissions.
Even if the submission isn't high-level, as long as it is written with care, we will accept it and provide feedback.

Recommended before submitting:
- Keep the scope of changes small.
- Include tests or verification steps if necessary.
- Avoid changes that only involve unrelated formatting.
- Clearly state any impact or risks to existing behavior.
- Write a concise summary and points to check.

Please understand that developers may not be able to review immediately.

### 4. PR Submission Checklist (Optional)
- Which issue does this change address?
- What problem are you trying to solve?
- Summary of changes.
- Verification steps.
- Presence of compatibility issues or side effects.
- Images or logs as needed.

### 5. Writing Bug Reports and Improvement Proposals
Please include reproduction steps, environment details (OS/Version/Locale), expected behavior, and actual behavior.
This information helps reduce the time until a fix is implemented.

---
Feel free to contribute. Don't worry about the barrier to entry; we welcome your proposals, PRs, Issues, and Discord reports.
